# 案件管理ツール 仕様書

## 1. 概要

本ツールは、個人事業主・小規模事業者が案件、タスク、請求状況、入金状況をローカル環境で管理するためのWebアプリケーションです。

Next.js上で動作し、データはSQLiteに保存します。案件ごとの請求書・納品書プレビューと印刷にも対応します。

## 2. 利用者

- 個人事業主
- フリーランス
- 小規模な受託制作・開発・運用業務の担当者

## 3. 主な機能

### 3.1 ダッシュボード

ダッシュボードでは、案件全体の状況を一覧できます。

- 総案件数の表示
- 進行中案件数の表示
- 今月納期の案件金額合計の表示
- 未入金案件の金額合計の表示
- 直近14日以内に納期がある未入金案件の表示
- ステータス別の案件一覧表示

### 3.2 案件一覧

案件一覧では、登録済み案件を検索・絞り込みできます。

- 案件名またはクライアント名でのキーワード検索
- 進捗ステータスでの絞り込み
- 請求ステータスでの絞り込み
- 入金ステータスでの絞り込み
- 納期範囲での絞り込み
- 案件カードから案件詳細へ遷移
- 未完了タスク数の表示

### 3.3 案件登録・編集

案件登録・編集では、案件の基本情報を管理できます。

入力項目:

- 案件名
- クライアント名
- 契約金額
- 納期
- 進捗ステータス
- 請求ステータス
- 入金ステータス
- ファイルURL
- 参考URL
- メモ

クライアント名は一意に管理され、同名クライアントが存在する場合は既存クライアントに紐づきます。

### 3.4 案件詳細

案件詳細では、案件の状態と関連タスクを確認・操作できます。

- 案件基本情報の表示
- 進捗、請求、入金ステータスの表示
- 納期、作成日、更新日の表示
- メモの表示
- ファイルURL、参考URLへの外部リンク表示
- 請求書プレビュー画面への遷移
- 納品書プレビュー画面への遷移
- 案件編集画面への遷移
- 案件削除

### 3.5 タスク管理

案件ごとにタスクを管理できます。

- タスク追加
- タスク完了・未完了の切り替え
- タスク削除
- タスク納期の管理
- タスクメモの管理

タスク一覧は、未完了タスクを先に表示し、同じ完了状態の中では納期昇順で表示します。

### 3.6 事業者設定

請求書・納品書に表示する事業者情報を管理できます。

入力項目:

- 屋号・事業者名
- 代表者名
- 郵便番号
- 住所
- 電話番号
- メールアドレス
- 請求書番号プレフィックス
- 消費税率
- 銀行名
- 支店名
- 口座番号
- 口座名義
- 帳票備考

設定は `BusinessSetting` の `default` レコードとして1件管理します。

### 3.7 請求書・納品書

案件詳細から、請求書・納品書のプレビュー画面を表示できます。

共通表示内容:

- 発行日
- 事業者情報
- クライアント名
- 案件名
- 案件金額
- 消費税
- 合計金額
- 備考

請求書固有の表示内容:

- 支払期限
- 振込先情報

納品書固有の表示内容:

- 納品内容

印刷ボタンにより、ブラウザの印刷機能を呼び出します。

## 4. ステータス定義

### 4.1 進捗ステータス

| 値 | 意味 |
| --- | --- |
| `NOT_STARTED` | 未着手 |
| `IN_PROGRESS` | 進行中 |
| `WAITING_REVIEW` | 確認待ち |
| `DELIVERED` | 納品済み |
| `INVOICED` | 請求済み |
| `PAID` | 入金済み |

### 4.2 請求ステータス

| 値 | 意味 |
| --- | --- |
| `NOT_BILLED` | 未請求 |
| `BILLING_PLANNED` | 請求予定 |
| `BILLED` | 請求済み |

### 4.3 入金ステータス

| 値 | 意味 |
| --- | --- |
| `UNPAID` | 未入金 |
| `PARTIAL` | 一部入金 |
| `PAID` | 入金済み |

## 5. 入力チェック

### 5.1 案件

- 案件名は必須、最大120文字
- クライアント名は必須、最大120文字
- 契約金額は0以上の整数
- ファイルURL、参考URLは入力された場合URL形式
- ステータスは定義済みenumのみ許可

### 5.2 タスク

- タスク名は必須、最大120文字
- 納期は任意
- メモは任意

### 5.3 事業者設定

- 事業者名、代表者名は最大120文字
- 郵便番号は最大20文字
- 住所は最大240文字
- 電話番号は最大40文字
- メールアドレスは入力された場合メール形式
- 請求書番号プレフィックスは最大20文字
- 消費税率は0から100の整数
- 帳票備考は最大500文字

## 6. 画面一覧

| URL | 画面 | 概要 |
| --- | --- | --- |
| `/` | ダッシュボード | 案件数、金額、納期が近い案件を表示 |
| `/projects` | 案件一覧 | 案件検索・絞り込み・一覧表示 |
| `/projects/new` | 案件新規作成 | 案件を登録 |
| `/projects/[id]` | 案件詳細 | 案件情報とタスクを表示・操作 |
| `/projects/[id]/edit` | 案件編集 | 案件を更新 |
| `/projects/[id]/invoice` | 請求書 | 請求書プレビュー・印刷 |
| `/projects/[id]/delivery-note` | 納品書 | 納品書プレビュー・印刷 |
| `/settings` | 設定 | 事業者情報と帳票設定を更新 |

## 7. 現状の制約・注意点

- 認証・ユーザー管理はありません。
- データはローカルSQLiteに保存されます。
- 請求書番号プレフィックスは設定できますが、現状のプレビューでは帳票番号の採番・表示には使われていません。
- `Invoice` と `DeliveryNote` のDBモデルはありますが、現在の画面は案件情報から帳票を即時プレビューしており、帳票レコードの作成・保存は行っていません。
